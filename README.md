# Wlameshot

A modern, Wayland-compatible screenshot tool inspired by Flameshot. (Wayland + Flameshot = **Wlameshot**.)

> **Note:** This project was generated by Claude (Anthropic's AI) under human
> direction. It is provided as-is; please review the code before relying on it.

## Features

- ✅ Wayland support (Qt QScreen with a `grim` fallback for wlroots/GNOME)
- ✅ GUI capture mode with rectangle selection
- ✅ Annotations: arrows, circles, rectangles, freehand pen, text, highlighter, numbered markers, blur
- ✅ Color picker (preset swatches + custom)
- ✅ Resizable/movable selection with handles
- ✅ Floating toolbar plus keyboard shortcuts
- ✅ Fullscreen capture
- ✅ Copy to clipboard and save to PNG

## Building

```bash
./build.sh                 # configure + build into build/
sudo make -C build install # optional: install the `wlameshot` binary

# ...or manually
mkdir build && cd build
cmake .. -DCMAKE_INSTALL_PREFIX=/usr/local
make -j$(nproc)
```

### Dependencies

Build:

- CMake >= 3.22
- Qt6 >= 6.5.0 (Core, Widgets, Gui)
- C++17 capable compiler (GCC 11+, Clang 14+)

Runtime (Wayland only — see [How It Works](#how-it-works)):

- `grim` and `wl-clipboard`

## Usage

```bash
# Launch GUI capture mode
wlameshot --gui

# Capture fullscreen
wlameshot --full

# Capture specific screen
wlameshot --screen 0
```

### GUI mode: annotating

In `--gui` mode, drag to select a region (drag the handles to resize, or drag
inside — with no tool active — to move it). A toolbar appears with drawing
tools; pick a color, draw inside the selection, then copy or save.

| Action | Shortcut |
| --- | --- |
| Arrow | `A` |
| Circle | `C` |
| Rectangle | `R` |
| Freehand pen | `P` |
| Text (click, type, `Enter` to commit) | `T` |
| Highlight | `H` |
| Numbered marker (click to place) | `N` |
| Blur / pixelate | `B` |
| Undo last annotation | `Ctrl+Z` |
| Save to PNG | `Ctrl+S` |
| Copy to clipboard | `Enter` |
| Cancel | `Esc` |

Colors are chosen from the toolbar swatches or the 🎨 custom-color picker.

### Waybar button (Hyprland / wlroots)

Add a launcher to Waybar. Reference `custom/screenshot` in a `modules-*` array,
then define it (uses the `nf-md-fire` glyph from a Nerd Font):

```jsonc
"custom/screenshot": {
  "format": "󰈸",
  "on-click": "wlameshot --gui",
  "on-click-right": "wlameshot --full",
  "tooltip-format": "Wlameshot — screenshot tool"
}
```

Restart Waybar to apply (`omarchy restart waybar`, or `killall -SIGUSR2 waybar`).

### Print Screen keybind (Hyprland)

Add to `~/.config/hypr/bindings.conf`. The `unbind` overrides any existing
`Print` binding (e.g. omarchy's default screenshot tool):

```
unbind = , PRINT
bindd = , PRINT, Screenshot (Wlameshot), exec, wlameshot --gui
```

Hyprland auto-reloads on save; check with `hyprctl configerrors`.

## How It Works

Capture uses Qt 6's `QScreen::grabWindow(0)` first:

```cpp
QScreen *screen = QGuiApplication::primaryScreen();
QPixmap pixmap = screen->grabWindow(0);
```

This works on X11, but **not** on most Wayland compositors (wlroots-based ones
such as Hyprland and Sway, as well as GNOME), where `grabWindow` returns a null
image for security reasons. When that happens the app falls back to the
[`grim`](https://sr.ht/~emersion/grim/) CLI to perform the capture, then feeds
the resulting image into the same selection/clipboard pipeline.

Similarly, on Wayland the clipboard is served via `wl-copy` (from `wl-clipboard`)
so the copied image survives after the process exits; on X11 the Qt clipboard is
used directly.

### Runtime dependencies on Wayland

- `grim` — screen capture (wlroots compositors)
- `wl-clipboard` (`wl-copy`) — clipboard persistence

On Arch: `sudo pacman -S grim wl-clipboard`
